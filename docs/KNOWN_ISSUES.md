# Known Issues & Future Improvements

Last reviewed: June 2026

## Known Issues

### 1. Placeholder page metadata

**Location:** `app/layout.tsx`

The site title and description still use the default `create-next-app` values (`"Create Next App"`, `"Generated by create next app"`). This affects browser tabs, search engine indexing, and social sharing previews.

**Fix:** Update `metadata` export with personal branding and a concise description.

---

### 2. Broken navbar anchor links

**Location:** `components/Navbar.tsx` â†’ `components/Timeline.tsx`, `components/Domains.tsx`

The navbar links to `#timeline` and `#domains`, but those section elements do not have matching `id` attributes. Clicking "Experience" or "Skills" in the nav does not scroll to the intended section.

| Nav link | Expected ID | Component | Has ID? |
|----------|-------------|-----------|---------|
| Projects | `projects` | `Projects` | Yes |
| Experience | `timeline` | `Timeline` | No |
| Skills | `domains` | `Domains` | No |
| Contact | `contact` | `Footer` | Yes |

**Fix:** Add `id="timeline"` to the Timeline section and `id="domains"` to the Domains section.

---

### 3. Project cards are not clickable

**Location:** `components/ProjectCard.tsx`, `data/projects.ts`

Project cards display title, description, and stack tags but have no links to live demos, GitHub repositories, or case study pages.

**Fix:** Add `url` and/or `repo` fields to project data and render CTA links in `ProjectCard`.

---

### 4. No mobile navigation menu

**Location:** `components/Navbar.tsx`

Navigation links are hidden on screens smaller than `md` (`hidden md:flex`). Mobile users cannot access section links from the navbar.

**Fix:** Add a hamburger menu or always-visible compact nav for mobile viewports.

---

### 5. SkillOrbit section has no nav link

**Location:** `components/SkillOrbit.tsx`, `components/Navbar.tsx`

The **Technical Universe** skills section appears on the page but is not linked from the navbar and has no section `id`.

**Fix:** Either add a nav link and `id`, or merge with the Domains section to avoid redundancy.

---

### 6. Content split across data files and components

Hero text, architecture layers, and contact links are hardcoded in components while other content lives in `data/`. This makes updates inconsistent.

**Fix:** Centralize profile, architecture, and contact data in `data/` files.

---

### 7. Unused `styles/` directory at workspace root

The parent workspace directory (`OneDrive/Projects/portfolio/`) contains empty placeholder folders (`app/`, `components/`, `data/`, etc.) alongside the actual project in `portfolio/portfolio/`. This is likely a OneDrive sync artifact.

**Fix:** Clean up the workspace root or add a pointer README explaining the nested structure.

---
